# User Personas (Draft)

Dokumen ini adalah draft awal untuk Phase 0. Detail final harus diisi setelah wawancara restoran (target minimal 5 restoran, lihat [ROADMAP.md](ROADMAP.md)).

## 1. Restaurant Owner — "Pak/Bu Pemilik"

- Profil: pemilik warung/restoran/cafe skala kecil-menengah, sering merangkap sebagai pengelola operasional harian.
- Masalah saat ini: menu cetak/PDF mahal untuk diganti tiap ada perubahan harga atau stok habis.
- Tujuan: menu selalu update, terlihat profesional, mudah dibagikan lewat QR.
- Kekhawatiran: kerumitan teknis, biaya bulanan, ketergantungan pada internet/HP.
- Pertanyaan validasi terkait: lihat bagian 8 [FEATURES_AND_PRICING.md](FEATURES_AND_PRICING.md).

## 2. Restaurant Staff — "Kasir/Karyawan"

- Profil: staf yang ditugaskan update menu sehari-hari (stok habis, harga).
- Kebutuhan: interface sederhana, cepat, tidak butuh training lama.
- Batasan: tidak boleh mengubah data sensitif (paket, kepemilikan) — lihat role di [FLOW.md](FLOW.md).

## 3. Customer — "Pelanggan Restoran"

- Profil: pengunjung yang scan QR di meja/kasir.
- Kebutuhan: menu terbuka cepat di HP, jelas kategori dan harga, mudah hubungi restoran (WA/lokasi).
- Tidak punya akun, tidak ingin proses ribet.

## 4. Super Admin — "Tim FTS"

- Profil: internal tim FTS yang mengelola seluruh tenant.
- Kebutuhan: visibilitas semua restoran, verifikasi pembayaran, monitoring penyalahgunaan.
